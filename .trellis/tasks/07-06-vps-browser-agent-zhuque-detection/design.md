# Design: VPS Browser-Agent Zhuque Detection

## Summary

Zhuque detection on VPS must no longer depend on server-side headless Playwright as the primary path. Instead, a paired Chrome extension running in the user's local browser executes Zhuque detection and returns normalized results to the VPS.

Local deployments keep the existing local browser transport. The new browser-agent transport is selected for VPS/Docker deployments through configuration.

## Transport Selection

Add a single transport selector:

```env
ZHUQUE_DETECT_TRANSPORT=auto
```

Allowed values:

| Value | Meaning | Intended deployment |
| --- | --- | --- |
| `auto` | choose from environment; local desktop -> `local_browser`, production Docker/VPS -> `browser_agent` when configured | default |
| `local_browser` | existing system Chrome/Edge/Brave CDP/local visible browser path | local Windows/WSL/Linux desktop |
| `browser_agent` | Chrome extension executes Zhuque detection on user's local machine | VPS/Docker recommended |
| `server_headless` | legacy Playwright/Chromium server-side detection | explicit fallback only |

Additional config:

```env
ZHUQUE_SERVER_HEADLESS_FALLBACK=false
ZHUQUE_BROWSER_AGENT_JOB_TIMEOUT=900
ZHUQUE_BROWSER_AGENT_HEARTBEAT_TIMEOUT=120
ZHUQUE_BROWSER_AGENT_PAIRING_TTL_SECONDS=600
ZHUQUE_BROWSER_AGENT_LONG_POLL_SECONDS=25
```

Rules:

- `browser_agent` mode must not silently fall back to `server_headless` unless `ZHUQUE_SERVER_HEADLESS_FALLBACK=true`.
- Local `python main.py` should continue to work through `local_browser`/current behavior.
- Docker/VPS docs should recommend `browser_agent`.

## Architecture

```text
User Chrome
  ├─ GankAIGC web app tab
  ├─ GankAIGC Chrome extension
  └─ Zhuque tab: https://matrix.tencent.com/ai-detect/
       │
       │ HTTPS long-poll / API calls
       ▼
VPS app container
  ├─ browser-agent routes
  ├─ workspace APIs
  └─ session/SSE APIs
       │
       ▼
PostgreSQL
  ├─ browser_agent_pairings
  ├─ browser_agents
  └─ zhuque_agent_jobs
       ▲
       │
VPS worker container
  └─ OptimizationService / ZhuqueService / BrowserAgentZhuqueTransport
```

The database is the cross-process handoff between app and worker. Do not use in-memory queues for browser-agent jobs.

## Data Flow

### Pairing

```text
Workspace -> POST /api/browser-agent/pairings
Backend -> create short-lived pairing code
User enters code in extension
Extension -> POST /api/browser-agent/claim
Backend -> verifies code, creates/updates browser_agents, returns agent_token
Extension stores agent_token in chrome.storage.local
Extension heartbeats periodically
Workspace polls agent status and shows online/offline
```

### Detection

```text
OptimizationService needs Zhuque detect(text)
BrowserAgentZhuqueTransport checks online agent
Transport creates zhuque_agent_jobs row
Extension long-polls /jobs/claim
Extension claims job
Extension opens/reuses Zhuque tab
Extension fills text and starts detection
Extension observes result or manual verification
Extension sends progress/complete/fail
Transport waits on DB status until complete/fail/timeout
OptimizationService continues existing reduce pipeline
```

## Database Tables

### `browser_agent_pairings`

Purpose: short-lived pairing codes generated by an authenticated web user.

Fields:

```text
id                   integer primary key
user_id              integer not null index
pairing_code_hash    string not null unique
expires_at           datetime not null
claimed_at           datetime nullable
claimed_by_agent_id  string nullable
created_at           datetime not null
```

Notes:

- Store only a hash of the pairing code.
- Pairing code TTL defaults to 10 minutes.
- A claimed or expired pairing cannot be reused.

### `browser_agents`

Purpose: persistent browser extension identity for a GankAIGC user.

Fields:

```text
id                   integer primary key
user_id              integer not null index
agent_id             string not null unique
name                 string nullable
token_hash           string not null
status               string not null default 'offline'
last_seen_at         datetime nullable
created_at           datetime not null
updated_at           datetime nullable
revoked_at           datetime nullable
capabilities_json    json/text nullable
user_agent           string nullable
extension_version    string nullable
```

Status values:

```text
online
offline
revoked
```

Online calculation:

```text
online if revoked_at is null and last_seen_at >= now - ZHUQUE_BROWSER_AGENT_HEARTBEAT_TIMEOUT
otherwise offline
```

The stored `status` is a cache/display aid; API status should compute freshness from `last_seen_at`.

### `zhuque_agent_jobs`

Purpose: durable work item that transfers a Zhuque detection request from worker to user extension and result back to worker.

Fields:

```text
id                     integer primary key
job_id                 string not null unique
user_id                integer not null index
session_id             integer nullable index
segment_id             integer nullable index
status                 string not null index
payload_text           text not null
payload_hash           string not null
result_json            json/text nullable
progress_json          json/text nullable
error_code             string nullable
error_message          text nullable
claimed_by_agent_id    string nullable index
created_at             datetime not null
claimed_at             datetime nullable
started_at             datetime nullable
completed_at           datetime nullable
expires_at             datetime not null
heartbeat_at           datetime nullable
attempt_count          integer not null default 0
```

Status values:

```text
pending
claimed
running
manual_required
completed
failed
expired
cancelled
```

Payload privacy:

- `payload_text` is required for MVP because the extension must receive the text.
- Avoid writing `payload_text` to application logs.
- Long-term hardening may encrypt the payload at rest using the existing app encryption key.

## API Contracts

All routes use `/api/browser-agent` prefix.

### Web-user routes

#### `POST /pairings`

Authenticated GankAIGC user creates a pairing code.

Response:

```json
{
  "pairing_id": 12,
  "pairing_code": "GANK-7K29",
  "expires_at": "2026-07-06T12:00:00Z"
}
```

Validation:

- User must be logged in.
- Old unclaimed pairings may be expired/ignored.

#### `GET /status`

Authenticated web user gets browser-agent status.

Response:

```json
{
  "required": true,
  "transport": "browser_agent",
  "online": true,
  "agents": [
    {
      "agent_id": "agent_abc",
      "name": "Chrome on Windows",
      "status": "online",
      "last_seen_at": "2026-07-06T12:00:00Z",
      "extension_version": "0.1.0"
    }
  ],
  "message": "本机浏览器插件在线"
}
```

#### `POST /revoke`

Authenticated user revokes one agent.

Request:

```json
{"agent_id": "agent_abc"}
```

Response:

```json
{"ok": true}
```

### Extension routes

Extension routes authenticate with:

```text
Authorization: Bearer <agent_token>
```

#### `POST /claim`

Claim a pairing code.

Request:

```json
{
  "pairing_code": "GANK-7K29",
  "agent_id": "agent_abc",
  "name": "Chrome on Windows",
  "extension_version": "0.1.0",
  "capabilities": {
    "zhuque_detect": true,
    "manual_verification": true
  }
}
```

Response:

```json
{
  "agent_token": "opaque-token",
  "agent_id": "agent_abc",
  "user_id": 1,
  "heartbeat_interval_seconds": 10,
  "job_poll_seconds": 25
}
```

#### `POST /heartbeat`

Request:

```json
{
  "agent_id": "agent_abc",
  "status": "online",
  "active_job_id": "job_xyz"
}
```

Response:

```json
{
  "ok": true,
  "server_time": "2026-07-06T12:00:00Z"
}
```

#### `POST /jobs/claim`

Long-poll for the next pending job.

Request:

```json
{
  "agent_id": "agent_abc",
  "wait_seconds": 25
}
```

Response when a job is available:

```json
{
  "job": {
    "job_id": "job_xyz",
    "kind": "zhuque_detect",
    "text": "...paper text...",
    "timeout_seconds": 180,
    "session_id": 123,
    "segment_id": null
  }
}
```

Response when no job:

```json
{"job": null}
```

Claim must be atomic: use DB row lock / skip-locked semantics.

#### `POST /jobs/{job_id}/progress`

Request:

```json
{
  "status": "running",
  "message": "正在本机朱雀页面检测",
  "progress": 0.35,
  "metadata": {
    "url": "https://matrix.tencent.com/ai-detect/"
  }
}
```

Allowed status updates:

```text
running
manual_required
```

Manual-required request:

```json
{
  "status": "manual_required",
  "message": "请在本机朱雀页面完成验证码",
  "metadata": {
    "reason": "captcha"
  }
}
```

#### `POST /jobs/{job_id}/complete`

Request:

```json
{
  "result": {
    "success": true,
    "source": "browser_agent",
    "rate": 61.35,
    "labels_ratio": {"0": 0.6135, "1": 0, "2": 0.3865},
    "segment_labels": [],
    "raw_payload": {}
  }
}
```

Response:

```json
{"ok": true}
```

#### `POST /jobs/{job_id}/fail`

Request:

```json
{
  "error_code": "zhuque_not_logged_in",
  "message": "请先在本机 Chrome 登录朱雀",
  "retryable": true
}
```

Response:

```json
{"ok": true}
```

## State Machines

### Browser agent state

```text
unpaired
  -> pairing_code_created
  -> paired_offline
  -> online
  -> busy
  -> online
  -> offline
  -> revoked
```

Transitions:

| From | Event | To |
| --- | --- | --- |
| `unpaired` | web creates pairing | `pairing_code_created` |
| `pairing_code_created` | extension claims code | `paired_offline` then `online` on heartbeat |
| `online` | job claimed | `busy` |
| `busy` | job complete/fail/expired | `online` |
| `online`/`busy` | heartbeat timeout | `offline` |
| any non-revoked | user revoke | `revoked` |

### Zhuque browser-agent job state

```text
pending
  -> claimed
  -> running
  -> manual_required
  -> running
  -> completed

pending/claimed/running/manual_required
  -> failed
  -> expired
  -> cancelled
```

Transitions:

| From | Event | To |
| --- | --- | --- |
| `pending` | extension claims | `claimed` |
| `claimed` | extension starts tab work | `running` |
| `running` | captcha/login verification needed | `manual_required` |
| `manual_required` | user completes verification and extension resumes | `running` |
| `running` | extension returns result | `completed` |
| any non-terminal | extension reports failure | `failed` |
| any non-terminal | job timeout | `expired` |
| any non-terminal | session/user cancellation | `cancelled` |

Terminal states:

```text
completed
failed
expired
cancelled
```

Worker behavior:

- `completed`: normalize result and continue pipeline.
- `manual_required`: keep waiting, emit progress/SSE message.
- `failed`: fail current detect attempt with actionable message.
- `expired`: fail with timeout and mark session failed or retry according to existing pipeline rules.
- `cancelled`: stop task if user/admin cancelled session.

## Backend Service Boundaries

### New services

```text
app/services/browser_agent_service.py
```

Owns:

- pairing code generation/claiming.
- token hashing/verification.
- heartbeat update.
- online/offline computation.
- job creation, claim, progress, complete, fail, expiry.

```text
app/services/zhuque_browser_agent_transport.py
```

Owns:

- implementation of browser-agent detect(text).
- selecting online agent.
- creating job.
- waiting for job result.
- converting job result into existing Zhuque result shape.

### Existing services to adapt

```text
app/services/zhuque_service.py
app/services/zhuque_api.py
app/services/optimization_service.py
```

Guideline:

- Keep current local/server Playwright logic intact.
- Add transport selection at the service boundary; avoid scattering `if VPS` branches through optimization pipeline.
- Existing result normalization remains the source of truth for Zhuque labels/rates.

## Chrome Extension Design

### Files

```text
browser-extension/
├── manifest.json
├── background.js
├── content-zhuque.js
├── injected-zhuque.js
├── popup.html
├── popup.js
├── styles.css
└── README.md
```

### Permissions

Minimum permissions:

```json
{
  "permissions": ["storage", "tabs", "scripting", "alarms"],
  "host_permissions": [
    "https://matrix.tencent.com/*",
    "https://<gankaigc-domain>/*"
  ]
}
```

No `<all_urls>`.

### Background service worker

Responsibilities:

- store server URL and agent token.
- claim pairing code.
- send heartbeat.
- long-poll jobs.
- open/reuse Zhuque tab.
- communicate with content script.
- report progress/complete/fail.

### Content script

Responsibilities:

- run on `https://matrix.tencent.com/*`.
- receive job command from background.
- inject page-world script if needed.
- perform DOM operations: clear input, set text, click detect.
- bridge page-world observed network/result events back to background.

### Injected page-world script

Responsibilities:

- observe Zhuque result payloads from page context where content script isolation prevents direct access.
- hook fetch/XHR/WebSocket only for Zhuque result endpoints/frames.
- send compact result payload through `window.postMessage`.

## Frontend UX Design

Workspace Zhuque card must branch by transport:

### Local browser transport

Keep current local behavior and copy.

### Browser-agent transport

States:

| State | Main copy | Actions |
| --- | --- | --- |
| no paired agent | `本机浏览器插件未连接` | install extension, create pairing code |
| pairing active | `请在插件中输入配对码` | copy code, regenerate |
| offline | `插件离线，请打开 Chrome 并保持插件在线` | reconnect guide |
| online | `本机浏览器插件在线` | open Zhuque page, revoke |
| busy | `本机浏览器正在执行朱雀检测` | view session |
| manual_required | `请在本机朱雀页面完成验证码/登录验证` | focus/open Zhuque page |

Start behavior:

- If `browser_agent` required and no online agent: block start/preflight with actionable message.
- If online: start task normally.

Session detail/SSE:

- Show browser-agent progress events in Agent trace/progress panel.
- Show manual-required prompt without marking task failed until timeout.

## Security Notes

- Agent token is an opaque random secret; server stores only hash.
- Pairing code is short-lived and hash-stored.
- Extension can only claim jobs for its paired user.
- Job completion/failure endpoints verify token and job ownership.
- Full paper text should not appear in logs, trace, or browser-agent status API except the claimed job payload sent to the authorized extension.
- Extension host permissions are scoped.
- No public CDP, no remote desktop tunneling, no cookie extraction from unrelated sites.

## Compatibility / Rollback

- Local deployment rollback: set `ZHUQUE_DETECT_TRANSPORT=local_browser`.
- VPS emergency rollback: set `ZHUQUE_DETECT_TRANSPORT=server_headless`, accepting CAPTCHA risk.
- Browser-agent hard fail fallback can be disabled with `ZHUQUE_SERVER_HEADLESS_FALLBACK=false`.

## Validation Strategy

- Backend unit/integration tests for APIs/state transitions.
- Existing `test_zhuque_integration.py` remains green.
- Static frontend tests for workspace copy/state branches.
- Manual extension test with unpacked extension.
- Manual VPS test to confirm no server-side Playwright Zhuque detection launches under `browser_agent` mode.
