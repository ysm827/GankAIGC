# GankAIGC 成熟化需求路线图

本文档记录上线后继续完善项目的需求拆分。目标不是堆功能，而是优先补齐稳定运行、故障定位、用户体验和项目专业度。

## P0：上线运维闭环

1. 后台健康检查页
   - 显示应用版本、运行环境、配置文件位置。
   - 显示数据库连通状态。
   - 显示 worker/inline worker 状态、排队任务、处理中任务、最近心跳时间。
   - 显示 VPS 在线更新前置条件和当前可用性。

2. 备份状态展示
   - Docker 部署默认读取宿主机 `backups/` 目录。
   - 后台展示最近备份时间、文件大小、备份数量、保留天数。
   - 支持下载最近备份，便于新手在后台拿到数据库备份文件。
   - 未检测到备份目录时给出明确提示。

3. 管理员模型 API 测试连接
   - 管理后台系统配置中增加“测试连接”。
   - 支持测试润色、增强、感情润色、压缩模型。
   - 返回失败原因：缺少 API Key、Base URL 格式错误、认证失败、模型不存在、请求超时、服务商错误。
   - 测试接口只发送极短消息，不消耗大量额度。

## P1：用户体验增强

1. 任务失败原因优化
   - 把底层异常转换为用户能理解的提示。
   - 区分 API Key 错误、余额不足、模型不存在、频率限制、网络超时。

2. 后台新手引导
   - 首次进入后台显示上线 checklist。
   - 包含修改管理员密码、配置模型 API、创建邀请码、测试处理、确认备份。

3. 用户邀请关系面板
   - 展示管理员邀请和用户邀请。
   - 展示每个邀请码注册人数、邀请人、使用人。

## P2：运营与项目专业度

1. 兑换码批量生成与导出
   - 支持批量生成 10/50/100 个兑换码。
   - 支持复制或导出 CSV。

2. 公告功能
   - 后台发布公告。
   - 用户首页或工作台展示维护通知、模型切换通知、使用说明。

3. GitHub 项目模板
   - 增加 Bug 反馈模板、部署问题模板、功能建议模板。
   - README 保持简单，把详细运维文档拆到 `docs/`。

## 当前实施顺序

第一轮已实现 P0 的三个功能：后台健康检查页、管理员 API 测试连接、备份状态展示。

第二轮已实现 P1 的三个功能：任务失败原因友好化、后台上线检查清单、用户邀请关系面板。

下一轮优先进入 P2：兑换码批量生成与导出、公告功能、GitHub 项目模板。
